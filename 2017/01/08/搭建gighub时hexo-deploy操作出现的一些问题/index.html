github+hexo是一个很火的博客搭建方法，当然网络上的教程也是数不胜数，在这里就不多赘述，如果有需要可以点击这里，在本篇博客中，主要提出在这些教程中未提到而我却遇到的问题（hexo deploy中出现的错误），目的是方便以后查阅。

ERROR Deployer not found: git

必须先有这一步才可以执行潜在的 git push 操作，不然无法顺利的发布到github上去，自然也无法通过github默认给你的域名访问

Not a git repository (or any of the parent directories): .git

当我看到这一句的时候其实是一头雾水，以为是github新建仓库的时候出现了某种不可描述的错误，非也。

配置Deployment首先，你需要为自己配置身份信息，打开命令行，然后输入：

  1. git config –global user.name “yourname”
  2. git config –global user.email “youremail”
    当你输入完之后 执行 
    hexo d / hexo deploy
    如果成功了，好本文结束，皆大欢喜，
    如果还未成功， 那么还需要第三步
  3. 检查你的根目录下的文件夹:.deploy_git，重新再deploy就行了

水平有限，如果不能解决你的问题那还是去网上看看大佬们怎么说的吧。
