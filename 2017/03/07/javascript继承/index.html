默认模式（原型链继承）

将子构造函数的原型指向父构造函数的实例。

function P () {
thie.name = 123;
}
function C () {
this.age = 234;
}
c.prototype = new P();
缺点：

包含引用类型值的属性会被所有原型共享（会有被修改的风险）
同时继承了两个对象的属性，既添加到this的属性以及原型的属性。在绝大时候不需要这些自身属性。
不支持将参数传入父构造函数
借用构造函数

就是简单在子类的构造函数中调用父类的构造函数。（借助于call）

function P () {
this.color = [1,3,4];
}
function C () {
P.call(this);
}
new C();
相当于在子构造函数中运行了父构造函数的代码而已。这里茶播一条new的过程：

当我们使用new生成一个新对象时，它会有以下流程： 1. 开辟一块新的内存空间； 2. 将空对象的原型（隐式指针proto）指向了构造函数的prototype属性。 3. 将空对象赋值给构造函数的this指针（也就是为什么当我们新建对象的时候this指向这个新建的对象） 4. 开始执行构造函数的代码
缺点：

并没有链接到父类的原型，在父类原型中定义的东西并不会被子类看到。
组合式继承

使用原型链实现对原型属性的和方法的继承，通过借用构造函数来实现对实例属性的继承。

function P () {
thie.name = 123;
}
P.prototype.sayName = function () {
alert(this.name);
}
function C () {
P.call(this,name);
}
c.prototype = new P();
注意： 因为这里先使用了P的构造函数又将P的实例作为了构造函数的原型。故对象本身的属性在对象的原型中也会同时拥有一套。

原型式继承

创建临时的构造函数然后将传入的对象作为这个构造函数的原型，最后返回这个临时类型的实例，这个跟默认的方法原理一致，而且子类并没有自身的变量。

寄生式继承

创建一个仅仅用于封装继承过程的函数，该函数在内部以某种方式来增强对象。

function createAnother (orifinal) {
var clone = object(original);
clone.sayHi = function () {
alert(“hi”);
}
}
寄生组合模式

上面说了组合模式的最大问题是会两次调用父类的构造函数。所谓的寄生组合式继承通过借用构造函数来继承属性，通过原型来继承方法。其背后的思路是：不必为了指定子类型的原型而调用父类构造函数，我们需要的无非是一个副本。本质上就是使用寄生式继承来继承父类的原型(而非父类的实例)。

function inherit (parent, child) {
var prototype = object(parent.prototype);
prototype.constructor = child
child.prototype = prototype;
}
关于上述object的用法

ECMAScript5新增规范，接受两个参数，第一个参数和Object.create()方法一样，第二个参数通过自己的描述定义，以这种方式指定的任何属性都会覆盖原型对象上同名属性。

ES6继承

在es6中新增加了创建类的方法 Class，将javascript的书写方式靠近java的写法。es6中的class可以看作是一个语法糖，他的绝大部分功能我们都可以通过es5来实现。构造函数的prototype属性在es6的类继续存在。
